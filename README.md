# Работа с потоками в C++: join() и detach()

Программа демонстрирует создание нескольких потоков, их параллельное выполнение, синхронизацию вывода в консоль и корректное завершение работы потоков.

## Структура проекта

- **part1** - работа с потоками с использованием `join()`.
- **part2** - использование `detach()` для одного потока и `join()` для другого.

## Реализовано

- Создание потоков с использованием `std::thread`.
- Параллельный вывод сообщений из нескольких потоков.
- Использование задержек между операциями (`std::this_thread::sleep_for`).
- Синхронизация доступа к `std::cout` с помощью `std::mutex`.
- Использование `std::lock_guard` для безопасной блокировки мьютекса.
- Управление завершением потоков с помощью:
  - `join()` (ожидание завершения потока);
  - `detach()` (отсоединение потока от основного).
- Демонстрация поведения программы при разных способах завершения потоков.

## Используемые функции и классы C++

| Функция / класс | Назначение |
|-----------------|------------|
| `std::thread` | Создание и управление потоками |
| `join()` | Ожидание завершения потока |
| `detach()` | Отсоединение потока от основного |
| `std::this_thread::sleep_for()` | Приостановка выполнения потока |
| `std::chrono::milliseconds` | Задание времени задержки |
| `std::mutex` | Синхронизация доступа к общим ресурсам |
| `std::lock_guard` | Безопасная блокировка мьютекса |
| `std::cout` | Вывод сообщений в консоль |

## Часть 1. Использование `join()`

Создаются два дочерних потока, которые выводят последовательность чисел с задержкой.  
Главный поток параллельно выводит собственные сообщения.

При использовании `join()` основной поток ожидает завершения всех дочерних потоков перед завершением программы.

### Анализ

- Если **не вызвать `join()` или `detach()`**, то при уничтожении объекта `std::thread` программа завершится аварийно (`std::terminate()`).
- При использовании `join()` главный поток **блокируется** и ждёт завершения дочернего потока.
- Порядок завершения потоков **не гарантируется** и зависит от планировщика потоков и времени выполнения.
- Программа **не завершится раньше времени**, так как `join()` обеспечивает ожидание завершения всех потоков.

## Часть 2. Использование `detach()`

В данной части один поток (`t1`) отсоединяется от основного с помощью `detach()`, а второй (`t2`) завершается с использованием `join()`.

Главный поток выполняет свою работу, затем ожидает завершения только потока `t2`.

### Анализ

- Если основной поток **завершится раньше отсоединённого**, выполнение `detach` потока может быть прервано.
- **Выполнение кода в отсоединённом потоке не гарантируется**, так как программа может завершиться раньше.
- Использование `detach()` может быть **опасным**, если:
  - поток использует ресурсы, которые будут уничтожены при завершении программы;
  - поток обращается к данным, принадлежащим основному потоку;
  - отсутствует контроль завершения потока.
